LATEST UPDATES + COMING SOON

Summary of updates

The next version of OpenEyes (v6.6) is due for release in the coming weeks and will bring a number of new features and improvements including: one-click op- notes (templates), a new Lacrimal Examination element, and a Pupil Distortion element.  

Longer term projects for the platform include the ability to record bilateral surgeries across all specialties, and a new mobile friendly OE interface that will allow for easier upload of photos to the patient record direct from your mobile. 

OpenEyes is currently on version v6.5.5

OpenEyes was updated to v6.5.5 in December 22. This version fixed a critical bug where Allergies and Alerts were not displaying in patient records.

KNOWN BUGS + WORKAROUNDS

Issues already queued to be fixed

  • Be sure to check that the PRINT box is ticked before saving/sending patient letters. Once it's saved, secreteries aren't able to print the letters unless the box has been ticked when saving. 
  • Copying information in Anterior Segment doodles from before 8th August causes Examinations to crash with red banner error. Do not copy these to avoid the error.
  • Certain free text boxes in Examination events do not automatically expand. Please send any that you find which do not expand as tickets to the IT service desk and this can be fixed.
  • When sending internal referrals, the 'Context' field is important, but not currently mandatory. Check it's complete before sending.
